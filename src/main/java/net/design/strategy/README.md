# 策略模式
## 一、定义接口类
ISuperCasher
- 声明统一的抽象方法 (getResult)

## 二、定义具体策略类
NormalCasher、DiscountCasher、ReturnCasher
- 实现抽象方法 (getResult)

## 三、定义上下文类
CashContext
- 供客户端调用，通过构造函数传入具体策略类

## 四、客户端调用
StrategyClient
- 调用上下文类，传入具体策略类，创建策略类对应的对象
- 调用策略类对应对象的方法，获取结果

## 五、简单工厂与策略类结合
FactoryClient
- 通过工厂类创建策略类对应的对象
- 调用策略类对应对象方法，获取结果

## 六、简单工厂与策略模式的区别
- 简单工厂模式：由工厂类根据客户端的传参创建对应的对象
- 策略模式：由客户端根据自身情况选择对应的策略类，创建对应的对象
- 策略类与客户端是多对多的关系，策略类与工厂类是多对一的关系
- 可通过策略类与工厂类结合，实现客户端不负责策略的选择
